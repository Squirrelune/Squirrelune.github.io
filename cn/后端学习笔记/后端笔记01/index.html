Maven

Maven 是一个基于项目对象模型（POM）的概念的纯java开发的开源的项目管理工具。它主要用于管理Java项目，进行依赖管理（jar包依赖管理）和项目构建（项目编译、打包、测试、部署）。此外，Maven还能分模块开发，提高开发效率。

核心特性

Maven 的一个核心特性是依赖管理。
传统的 Web 项目中，我们必须将工程所依赖的 jar 包复制到工程中，导致工程变得很大。Maven 工程中不直接将 jar 包导入到工程中，而是通过在 pom.xml 文件中添加所需 jar 包的坐标，这样就很好地避免了 jar 直接引入进来。在需要用到 jar 包时，只要查找 pom.xml 文件，再通过 pom.xml 文件中的坐标，到一个专门用于“存放 jar 包的仓库”（maven 仓库）中根据坐标找到这些 jar 包，再把这些 jar 包拿去运行。

基本要素：
project 这是所有 Maven pom.xml 文件中的顶级元素。
modelVersion 此元素指示此 POM 使用的对象模型的版本。
groupId 此元素指示创建项目的组织或组的唯一标识符。
artifactId 此元素指示此项目正在生成的主要工件的唯一基本名称。
version 此元素指示项目生成的工件的版本。
name 此元素指示用于项目的显示名称。
url 此元素指示可以找到项目站点的位置。
properties 此元素包含可在 POM 中的任何位置访问的值占位符。
dependencies 此元素的子项列出依赖项。POM 的基石。
build 这个元素处理诸如声明项目的目录结构和管理插件之类的事情。

Spring Boot

特性

“约定大于配置”思想：通过提供默认配置和标准化约定，减少开发者的手动配置工作。它自动配置应用程序、遵循标准项目结构、简化依赖管理和注解使用，使开发者能够快速启动和运行项目，专注于业务逻辑，而无需处理繁琐的配置细节。
IOC（控制反转）思想：将原本由代码直接操控的对象的调用权交给Spring容器来管理，以解耦代码，提高程序的可维护性和可扩展性。本该由我们管理的对象，Spring Boot可以替我们管理，这里的管理包括了创建、使用、销毁等过程。
